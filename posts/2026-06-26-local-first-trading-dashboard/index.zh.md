# 用 FastAPI + React 做一个本地优先、只读的交易分析面板

> **先把话说清楚，免得误会：** 这是一篇讲*软件工程*的文章，不是讲投资。这个项目是个**只读的分析面板**：它不下单、不发买卖信号、不承诺任何收益。它显示的每个数字，要么是历史统计、要么是确定性公式。本文不构成任何投资建议。

## 开头

我想要一块屏幕，把我做决定前要看的东西全拢到一起——持仓、延迟报价、几个技术指标、一些宏观背景、一份新闻摘要，还有我自己写下的"交易法则"——然后帮我*思考*，而不是替我*操作*。我给自己定的硬约束对一个交易工具来说有点反常：**本地优先、默认只读。** 这个程序应该能整天杵在那儿、什么都不碰，只有我明确按下按钮时才连券商——而且即便连上，也只*读*。

这篇讲这个约束怎么塑造了架构，以及一路上我觉得有意思的工程细节。

---

## 1. 只读是设计出来的，不是承诺出来的

我最在意的安全属性，被放在好几个互相独立的地方强制执行，这样它不会被"忘掉"：

- **默认根本不连券商。** 首页、轮询循环、一键组合计划——全都跑在一份只读的本地快照 + 免费行情上。只有显式发起刷新请求，才会去连 Interactive Brokers。
- **真要连，也是只读地连。** 每个券商会话都用 `readonly=True` 打开，而且**整个代码库里没有任何下单调用**——它永远只读持仓和账户摘要。
- **它宁可降级也不撒谎。** 网关离线时，它不回退假数据；而是拿外部行情给"最近一次真实快照"重新估值，并标注估值模式（`live` / `snapshot_external_mark` / `snapshot_stale`），让你永远知道数字有多新。

只读不是 README 里的一句话——它是代码的形状。

## 2. 数据诚实度：宁可"别用"，也不给你一个好看的假数

这个项目最有脾气的地方，是它宁可什么都不给你，也不给一个看起来对、其实不对的数：

- 如果一个报价的买卖价差超过合理阈值（或买价 > 卖价），这个值会被标为失真，界面直接写"失真，勿用"。
- 如果某个期权的隐含波动率落在合理区间之外，就判定为坏——据它算出的 Greeks 会被**弃用**，而不是带着虚假的自信显示出来。
- 付费数据有硬闸：取实时盘口（按次小额收费）要在界面里显式确认，服务端再加一层节流，防止双击重复扣费。
- 期权 Greeks 取不到时，代码会说明*为什么*——"未订阅实时期权数据源"——并回退到延迟数据 + 本地估算，而不是默默显示空白。

一个你没法一眼信任的面板，比没有面板更糟。所以它花了真功夫去*证明*自己的数字。

## 3. 技术栈（以及刻意没有的东西）

- **前端**：Vite 7 + React 19 + TypeScript，依赖列表刻意极简——`react`、`react-dom`、一套图标、一个 Markdown 渲染器。**没有任何图表库。** 这里的"可视化"是结构化面板、表格、状态条，不是 K 线图——这恰好契合一个关于*读懂背景*、而非*盯着价格*的工具。
- **后端**：FastAPI + uvicorn，实际只有三个 Python 依赖；其余几乎全靠标准库（HTTP、CSV、cookie、并发）。最近刚从一个大文件重构成十来个聚焦的模块。
- **数据分层（按成本）**：默认免费（延迟报价、技术指标、基本面、期权链 IV，加宏观数据和经济日历）；取实时盘口按次小额收费；实时期权 Greeks 需月费订阅（默认关，自动回退）。

## 4. 不靠 WebSocket 的"实时"

这里没有 socket 管线——只有克制的轮询 + 多级缓存。前端每隔几秒探一次券商连通性和组合、用更慢的节奏刷宏观；后端给每个源各自的 TTL（行情按秒、期权按分、基本面和日线按小时、日历按天）。决策端点把七八个跨国抓取丢进线程池并行，于是总延迟≈最慢的那一个源、而不是它们之和，再用一个全局信号量给并发封顶，免得免费报价 API 开始返回 429。

## 5. 确定性数学，不是预测

"数学期望"面板是对历史的纯算术，并且到处都标明了这一点：

- 盈亏平衡胜率就是 `风险 /（风险 + 回报）`。
- 历史命中率"回测"走两年日线，把每一天当作入场点，问在固定 horizon 内价格是先到目标还是先到止损——同日双触保守地记为亏。
- 它能对当下做条件化：只取与今天同一 RSI 桶、同趋势的历史日。

期权 Greeks 用本地 Black-Scholes 现算（输入延迟 IV，无风险利率取 3 个月美债）。这些都不预测任何东西——它是一种透明地*框定*决策的方式，这也正是为什么把它放上屏幕是安全的。

## 6. 对个人法则库的六阶段检索

最好玩的工程是法则检索。我的交易"法则"以笔记形式存在；程序给它们建索引，检索出相关的几条放进提示词。它是一条**provider 无关、可优雅降级**的管线：词法召回（对没有词边界的语言用 bigram 切词）→ 语义向量 → 多查询扩展 → section 扩展 → LLM 重排 → 全量复核。任一上层失败就退回纯词法——绝不崩。embedding 落盘缓存，签名带正文哈希，改笔记或换 provider 会自动重算。

还有一个很妙的逃生口：一个端点把整套法则导出成纯文本。因为法则库不大，你可以把整套粘进一个长上下文的网页版大模型，免费拿到全覆盖检索，完全不依赖服务端的 LLM key。

## 7. 踩坑故事

- **一个半开的网关把整页冻住。** 券商客户端的账户/组合调用在慢网关上*没有超时*——我实测干等约 54 秒，旧代码还全程占着锁、把一切都堵死。修法：单航班（single-flight）executor + 硬超时，到点先回缓存快照，让慢线程在后台把缓存暖好。
- **一个数据源悄悄死了。** 一家免费历史数据源的页面搬了家、然后长出了 JS 反爬挑战，于是我整体迁到另一个行情 API（它自己又需要 cookie + token 握手，每 30 分钟刷新）。
- **"期权陈旧"是误诊。** 一个"失真/不可用"的标记，结果是*休市时的宽价差*，不是数据陈旧——开盘就收窄。搞清这点后，我去加固了可信度判定，而不是去追一个根本不存在的 bug。
- **部署本身是个 boss 战。** 浏览器缓存让你以为没部署成功；`docker exec` 喂 stdin 必须带 `-i`；SSH 单命令参数有上限，逼得打包好的前端要 base64 切块传；git-bash 会把绝对路径搅乱，直到加 `MSYS_NO_PATHCONV=1`。

## 8. 诚实的取舍

- **它是单用户的个人工具。** CORS 钉死在 localhost，没有多租户鉴权（靠私网兜底），只有一份账户快照。不是产品。
- **免费数据是延迟的**（约 15 分钟）；真·实时需要付费订阅，多数 Greeks 是本地估算。
- **它依赖第三方非官方接口**，随时可能变——已经有一个变过了。
- **还有 MVP 残留**（mock 路由和样例数据）留在树里；代码比 README 的部分内容更超前。
- 以及最重要的那条再说一遍：**这是只读分析，不是自动交易，也不是建议。**

这里有意思的工作，从来都不是关于市场的。它是关于做一个对自己的数据诚实、构造上就安全、且真能帮你思考的工具——这三件事，放在任何面板上都值得在意。
